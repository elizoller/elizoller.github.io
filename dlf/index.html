DRS Project Toolkit Goals

  • Develop a lightweight and sustainable model for preserving and displaying/distributing project materials.
  • Provide Northeastern projects with a low-barrier customized website to share project work.
  • Support Northeastern projects by preserving project materials in the DRS.
  • Encourage collaborative development between Toolkit sites.

Suggested Features

  • Faceted full text search
  • Visualizations, particularly maps and timelines
  • Downloads
  • Sharing, including citations, social media, and object embedding
  • Easily interpreted data feed (i.e. JSON, CSV, RDF, XML)
  • Credit pages for project creators and curators
  • Metrics
  • Exhibits and/or galleries
  • Support for objects that are not stored in the DRS
  • Two-way syncing data between the web publishing tool and the DRS
  • Annotations
  • XML-aware search

DRS API

Written by David Cliff, Lead Digital Library Developer at Northeastern University Libraries

Single Item

/api/v1/search/[item pid]
retrieves MODS, derivatives, and master files

Collection Export

/api/v1/export/[collection pid]
retrieves MODS, derivatives, and master files for all items that are children of collection

Scoped Search

/api/v1/search/[collection pid]
retrieves Blacklight search results scoped with collection, accepts facets and keywords

DRS + Wordpress

a plugin for functionality and a child theme from an existing wordpress theme (quest) which provides many customization options and additional ones related to the drs content specifically

Search

Sets up a search page with list style and overrides wordpress search boxes with the search that hits the DRS API. Renders DRS results prominently and wordpress content minimally. Exposes facets and uses AJAX to update the page with fresh DRS results.

Browse

Sets up a browse page with a tiled styling. Hits the DRS API every time to get fresh results

Collection Browse

Allows users to browse by sub-collections of the collection entered in the settings page

Single Item

Provides a single item page retrieved dynamically from the DRS API with thumbnail or embedded media, MODS, and downloads of derivatives

Admin Settings

Allows users to input collection URL and then search/browse works automatically, allows customization of metadata on search/browse/collection pages and page titles
